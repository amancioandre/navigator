  <action>
Refactor `src/navigator/executor.py`. Keep `build_clean_env` and `build_command_args` unchanged. Replace `execute_command` and add new internals:

1. **Add ExecutionResult dataclass** at top of file:
   ```python
   @dataclass
   class ExecutionResult:
       command_name: str
       returncode: int
       stdout: str
       stderr: str
       attempts: int
       duration: float
       timed_out: bool
       log_path: Path | None
   ```

2. **Add _active_processes module-level set and cleanup** per D-16:
   - `_active_processes: set[int] = set()` -- tracks PIDs of active children
   - `_cleanup_children()` function: iterate `_active_processes`, call `os.killpg(os.getpgid(pid), signal.SIGTERM)` wrapped in try/except for ProcessLookupError/PermissionError
   - `atexit.register(_cleanup_children)`
   - Signal handlers for SIGTERM and SIGINT that call `_cleanup_children()` then re-raise with default handler
   - Guard signal registration with `threading.current_thread() is threading.main_thread()` per Pitfall 3

3. **Add _run_once(args, env, cwd, timeout) -> tuple[int, str, str]** per D-13, D-10, D-11:
   - Use `subprocess.Popen(args, env=env, cwd=cwd, stdout=PIPE, stderr=PIPE, text=True, start_new_session=True)` per D-13
   - Get `pgid = os.getpgid(proc.pid)` per D-15
   - Add PID to `_active_processes`
   - Call `proc.communicate(timeout=timeout)` per D-10
   - On success: remove PID from `_active_processes`, return `(returncode, stdout, stderr)`
   - On `TimeoutExpired`: SIGTERM the process group, wait 5s grace, SIGKILL if still alive per D-10. Return `(124, partial_stdout, partial_stderr)` per D-11
   - Always remove PID from `_active_processes` in finally block

4. **Refactor execute_command(cmd, config, timeout_override=None, retries_override=None) -> ExecutionResult** per D-01 through D-04:
   - Keep existing validation (claude CLI check, cwd check)
   - Load secrets, build env, build args (unchanged)
   - Determine `max_retries` from `retries_override or config.default_retry_count`
   - Determine `timeout` from `timeout_override or config.default_timeout`
   - Loop: first attempt (attempt=0), then up to `max_retries` additional attempts
   - On attempt > 0: compute delay as `2 ** attempt` seconds per D-01, log retry info per D-04, call `time.sleep(delay)`
   - Call `_run_once` for each attempt
   - After each attempt: call `write_execution_log` from execution_logger per D-05, D-06
   - Track `time.monotonic()` for total duration
   - If returncode == 0, break (success) per D-03
   - Return ExecutionResult with all fields populated

**New imports needed:** `atexit`, `os`, `signal`, `threading`, `time` from stdlib; `from dataclasses import dataclass`; `from navigator.execution_logger import write_execution_log`.
