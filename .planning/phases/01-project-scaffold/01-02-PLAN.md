  <action>
Create `src/navigator/config.py` implementing these components:

1. `get_config_dir() -> Path`: Returns `Path(platformdirs.user_config_dir("navigator"))`. Uses platformdirs, NOT hardcoded `~/.config` (per anti-pattern in research).

2. `get_data_dir() -> Path`: Returns `Path(platformdirs.user_data_dir("navigator"))`. DB and logs go in data dir (XDG convention: data is mutable, config is settings).

3. `get_config_path() -> Path`: Returns `get_config_dir() / "config.toml"`.

4. `resolve_path(path: str | Path) -> Path`: Calls `Path(path).expanduser().resolve()`. This is the INFRA-07 utility — used at registration time to prevent stale crontab refs.

5. `class NavigatorConfig(BaseModel)` with Pydantic v2:
   - `db_path: Path = Field(default_factory=lambda: get_data_dir() / "registry.db")`
   - `log_dir: Path = Field(default_factory=lambda: get_data_dir() / "logs")`
   - `secrets_base_path: Path = Field(default_factory=lambda: Path.home() / ".secrets" / "navigator")`
   - `default_retry_count: int = 3`
   - `default_timeout: int = 300`
   - Method `resolve_paths(self) -> None` that calls `resolve_path()` on db_path, log_dir, secrets_base_path and assigns back.

6. `load_config() -> NavigatorConfig`:
   - Get config_path from `get_config_path()`
   - If exists: read with `tomllib.load()`, construct `NavigatorConfig(**data)`
   - If not exists: create `NavigatorConfig()` with defaults, mkdir parents, serialize path fields to strings, write with `tomli_w.dump()`
   - Always call `config.resolve_paths()` before returning
   - Use `model_dump(mode="json")` for serialization — Pydantic v2 converts Path to string in json mode

Important: Import `tomllib` from stdlib (Python 3.12+). Import `tomli_w` for writing. Do NOT use `tomlkit`.
