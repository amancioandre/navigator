    - walk_chain(conn, start_name) returns list of Command objects in chain order (A -> B -> C)
    - walk_chain resolves cross-namespace names via parse_qualified_name
    - walk_chain raises ValueError if chain exceeds max_chain_depth (default 10)
    - detect_cycle(conn, from_name, to_name) returns True if adding from_name->to_name creates a cycle
    - detect_cycle returns False for valid non-cyclic links
    - detect_cycle detects self-links (A->A)
    - detect_cycle detects indirect cycles (A->B->C, adding C->A)
    - execute_chain(conn, start_cmd, config) runs each command sequentially via execute_command
    - execute_chain generates a UUID4 correlation ID (NAVIGATOR_CHAIN_ID)
    - execute_chain stops on first failure by default (stop-on-failure)
    - execute_chain continues past failures when on_failure_continue=True on the failed link
    - execute_chain returns a ChainResult with correlation_id, results list, and overall success
    - Each step in chain gets NAVIGATOR_CHAIN_ID injected via env (monkey-patch build_clean_env or pass extra_env)
  </behavior>

1. Create `src/navigator/chainer.py` with:

   ```python
   @dataclass
   class ChainResult:

       results: list[ExecutionResult]  # one per step

       steps_run: int
       total_steps: int
   ```

   - `walk_chain(conn, start_name, max_depth=10)`:
     - Start from start_name, follow chain_next links via get_command_by_name
     - For cross-namespace chain_next values, use parse_qualified_name to split ns:name, then look up by bare name
     - Collect commands into a list. Stop when chain_next is None
     - If len exceeds max_depth, raise ValueError("Chain depth limit exceeded: {max_depth}")
     - Return list[Command]

   - `detect_cycle(conn, from_name, to_name)`:
     - If from_name == to_name, return True (self-link)
     - Walk from to_name following chain_next links
     - If any link resolves to from_name, return True
     - If chain ends (None) without hitting from_name, return False

   - `execute_chain(conn, start_cmd, config)`:
     - Generate correlation_id = str(uuid.uuid4()) (per D-12)
     - Walk chain from start_cmd using walk_chain(conn, start_cmd.name, config.max_chain_depth)
     - For each command in chain:
       - Monkeypatch or wrap: inject NAVIGATOR_CHAIN_ID=correlation_id into the subprocess environment
       - The cleanest approach: temporarily set os.environ["NAVIGATOR_CHAIN_ID"] = correlation_id before calling execute_command, then remove it after. The build_clean_env function reads from os.environ for whitelisted vars, but NAVIGATOR_CHAIN_ID is not whitelisted. Instead, add "NAVIGATOR_CHAIN_ID" to ENV_WHITELIST temporarily, OR better: modify build_clean_env to accept an optional extra_env dict parameter. Update executor.py's build_clean_env signature to `def build_clean_env(secrets=None, extra_env=None)` and merge extra_env after secrets. Then update execute_command to accept and pass through extra_env.
       - Per D-09: each command fully completes before starting the next (sequential blocking)

       - Per D-05/D-06: if a step fails (returncode != 0) and cmd.on_failure_continue is False, stop the chain
       - Log chain start, each step result, and chain end via write_execution_log with correlation_id in the output (per D-14)
     - Return ChainResult
