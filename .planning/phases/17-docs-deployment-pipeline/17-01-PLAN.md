1. Create directory `.github/workflows/` if it does not exist.

2. Create `.github/workflows/docs.yml` with this exact content:

```yaml
name: docs

on:
  push:
    branches:
      - master

permissions:
  contents: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Configure Git credentials
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"

      - name: Install uv
        uses: astral-sh/setup-uv@v7
        with:
          enable-cache: true

      - name: Install dependencies
        run: uv sync --group docs --locked

      - name: Deploy docs
        run: uv run mkdocs gh-deploy --force --strict
```

3. Update `mkdocs.yml` line 3 — change `site_url: ""` to `site_url: https://amancioandre.github.io/navigator/`

This is the only change to mkdocs.yml. Do not modify any other lines.

Key details:
- `permissions: contents: write` is required for pushing to gh-pages branch (DEPLOY-01)
- `enable-cache: true` on setup-uv caches dependencies keyed by lockfile hash (DEPLOY-03)
- `--force` flag force-pushes gh-pages branch (safe — only generated output) (DEPLOY-02)
- `--strict` flag fails on mkdocs warnings, maintaining quality gate
- `--locked` flag ensures uv.lock is in sync with pyproject.toml
- GitHub Actions automatically creates commit status checks from workflow runs (DEPLOY-04)
- The `mkdocs gh-deploy` command auto-creates the gh-pages branch on first run (DEPLOY-02)
