1. **# Systemd Service** -- one-sentence overview: Navigator installs a systemd user service so the watcher daemon starts automatically and survives reboots.
2. **## Prerequisites** -- systemd-based Linux system (Ubuntu, Pop!_OS, etc.), at least one file watcher configured (link to file watching guide)
3. **## Install the Service** -- show `navigator install-service` with expected output. Explain what happens: unit file written to `~/.config/systemd/user/navigator.service`, daemon-reload, service enabled, `loginctl enable-linger` called. Show the `--no-linger` flag for systems where linger is already configured or not desired.
4. **## Control the Service** -- show all four actions:
   - `navigator service start`
   - `navigator service stop`
   - `navigator service restart`
   - `navigator service status` (show example status output)
5. **## Common Patterns** with two subsections:
   - **Verify reboot survival**: `navigator install-service`, then `sudo reboot`, then after reboot: `navigator service status` should show active
   - **Foreground mode for debugging**: `navigator daemon` runs the watcher in the foreground with visible log output, useful for troubleshooting before installing the service
6. **## Uninstall the Service** -- show `navigator uninstall-service`. Explain it stops the service, disables it, removes the unit file, and reloads the systemd daemon.
7. **## Unit File Details** -- show the generated unit file content:
   ```ini
   [Unit]
   Description=Navigator Watcher Daemon
   After=network.target

   [Service]
   Type=simple
   ExecStart={navigator_path} daemon
   Restart=on-failure
   RestartSec=5
   Environment=PYTHONUNBUFFERED=1

   [Install]
   WantedBy=default.target
   ```
   Explain: `Restart=on-failure` means the daemon restarts if it crashes, `RestartSec=5` waits 5 seconds before restart.
8. **## Troubleshooting**:
   - **Service not starting**: run `navigator daemon` in foreground to see error output, check `journalctl --user -u navigator` for logs
   - **Service does not survive reboot**: verify linger is enabled with `loginctl show-user $USER | grep Linger`, re-run `navigator install-service` without `--no-linger`
   - **Service running but watchers not triggering**: verify watchers are configured with `navigator watch --list`
9. **## What's Next** -- link to [File Watching](file-watching.md) for configuring watchers, link to [Scheduling](scheduling.md) for cron-based triggers that do not need the daemon
