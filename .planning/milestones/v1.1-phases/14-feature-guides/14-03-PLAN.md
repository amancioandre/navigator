| Option | Type | Default | Description |

| `log_dir` | path | `~/.local/share/navigator/logs` | Directory for execution log files |
| `secrets_base_path` | path | `~/.secrets/navigator` | Base directory for namespace secrets resolution |
| `default_retry_count` | integer | `3` | Number of retry attempts when a command fails |
| `default_timeout` | integer | `300` | Maximum execution time in seconds before a command is killed |
| `max_chain_depth` | integer | `10` | Maximum number of links in a command chain |

4. **## Example Config File** -- show a complete `config.toml` example:
```toml
db_path = "~/.local/share/navigator/registry.db"
log_dir = "~/.local/share/navigator/logs"
secrets_base_path = "~/.secrets/navigator"
default_retry_count = 3
default_timeout = 300
max_chain_depth = 10
```

5. **## Path Resolution** -- explain:
   - All paths are resolved to absolute paths on load
   - `~` is expanded to the user's home directory
   - Data directory follows XDG_DATA_HOME (defaults to `~/.local/share/`)
   - Config directory follows XDG_CONFIG_HOME (defaults to `~/.config/`)

6. **## Common Patterns** with two subsections:
   - **Increase timeout for long-running commands**: set `default_timeout = 600` for 10-minute timeout. Note individual commands can override with `--timeout` flag on register.
   - **Custom secrets location**: set `secrets_base_path = "/path/to/secrets"` to change where namespace auto-resolution looks for secrets files.

7. **## Troubleshooting**:
   - **Config not loading**: verify file exists at `~/.config/navigator/config.toml`, check TOML syntax is valid
   - **Paths not resolving**: ensure paths use forward slashes, `~` is supported for home directory
   - **Changes not taking effect**: Navigator reads config on each invocation, no restart needed

8. **## What's Next** -- link to [Secrets](secrets.md) for details on secret injection, link to [CLI Reference](../reference/cli.md) for per-command overrides of defaults
