3. **If the build fails, diagnose and fix.** Common issues from research:

   - **`site_url` warning:** If `site_url: ""` triggers a strict warning, change it to `site_url: "http://localhost:8000"` in mkdocs.yml.

   - **Module import error (`ModuleNotFoundError`):** Ensure `uv sync --group docs` was run (installs the project in editable mode). If the bridge module path is wrong, verify `navigator._click_bridge` is importable with `uv run python -c "from navigator._click_bridge import cli; print(type(cli))"`.

   - **mkdocs-click directive not processed:** Check plugin load order in mkdocs.yml. `search` must come before `mkdocs-click`. If the raw directive text appears in HTML output, the plugin is not loading.

   - **Nav/file mismatch:** Ensure every file listed in `nav:` exists in `docs/` and every `.md` file in `docs/` is listed in `nav:`.

4. **If mkdocs-click produces poor output for nested subcommands** (the namespace subgroup), test mkdocs-typer2 as a fallback:
