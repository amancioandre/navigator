9. **## What's Next** -- link to [Systemd Service](systemd.md) for running the watcher daemon persistently, link to [Scheduling](scheduling.md) for time-based triggers

Create `docs/guides/chaining.md` (GUIDE-03) with these exact sections:

1. **# Command Chaining** -- one-sentence overview: Navigator chains commands so one triggers the next on completion, with correlation tracking and cycle protection.
2. **## Prerequisites** -- two or more registered commands
3. **## Chain Commands** -- show `navigator chain build-project --next run-tests` with expected output. Explain chain stored as `chain_next` field. Show viewing the chain: `navigator chain build-project --show` with output `build-project -> run-tests`.
4. **## Common Patterns** with three subsections:
   - **Build-test-deploy pipeline**: chain three commands, show `--show` output: `build -> test -> deploy`
   - **Continue on failure**: `navigator chain build-project --next notify-team --on-failure continue` (explain default is `stop`)
   - **Cross-namespace chaining**: `navigator chain myproject:build --next myproject:deploy` (explain qualified name syntax `namespace:command`)
5. **## Correlation IDs** -- explain that Navigator generates a UUID4 correlation ID passed as `NAVIGATOR_CHAIN_ID` environment variable to every command in the chain. Useful for tracing log output across chained executions.
6. **## Cycle Detection** -- explain Navigator checks for cycles before linking. Self-links and transitive cycles (A->B->C->A) are rejected. Show error behavior.
7. **## Chain Depth Limit** -- explain `max_chain_depth` config option (default: 10). Chains exceeding this depth stop executing.
8. **## Remove a Chain Link** -- show `navigator chain build-project --remove`
9. **## Troubleshooting**:
   - **Chain stops unexpectedly**: check if a command failed (default is stop on failure), use `--on-failure continue` if appropriate
   - **Cycle detected error**: run `navigator chain cmd --show` to see the full chain and identify the loop
   - **Chain too deep**: increase `max_chain_depth` in config.toml (link to configuration guide)
10. **## What's Next** -- link to [Namespaces](namespaces.md) for organizing commands across projects, link to [Configuration](configuration.md) for max_chain_depth setting

Use `!!! tip` admonition for: "Use `navigator chain cmd --show` to visualize the full chain before adding new links."
