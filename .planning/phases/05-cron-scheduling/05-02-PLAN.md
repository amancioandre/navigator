**Function signature (per D-08, D-09, D-10, Pitfall 6):**
```python
@app.command()
def schedule(
    command: Annotated[
        str | None,
        typer.Argument(help="Command name to schedule"),
    ] = None,
    cron_expr: Annotated[
        str | None,
        typer.Option("--cron", help='Cron expression (e.g., "*/5 * * * *")'),
    ] = None,
    remove: Annotated[
        bool,
        typer.Option("--remove", help="Remove schedule for command"),
    ] = False,
    list_all: Annotated[
        bool,
        typer.Option("--list", help="List all scheduled commands"),
    ] = False,
) -> None:
```

**Implementation logic:**

1. **--list mode (D-10):** If `list_all` is True:
   - Lazy import: `from navigator.scheduler import CrontabManager` and `from navigator.config import get_data_dir`
   - Create `CrontabManager(lock_path=get_data_dir() / "crontab.lock")`
   - Call `manager.list_schedules()`
   - If empty: `console.print("[dim]No scheduled commands.[/dim]")`
   - Otherwise: Rich Table with columns "Command" (cyan), "Schedule" (green), "Enabled" (dim)
   - Return early (command argument not needed)

2. **Validation:** If `command` is None (and not list_all): print error "Command name required", exit 1
   - If `cron_expr` and `remove` both specified: print error "Cannot use --cron and --remove together", exit 1
   - If neither `cron_expr` nor `remove`: print error "Use --cron to schedule or --remove to unschedule", exit 1

3. **Command exists check:** Lazy import db functions, load config, get connection, check `get_command_by_name(conn, command)`. If None: print red error, exit 1. If cmd.status == "paused": print red error suggesting `navigator resume`, exit 1. Close connection in finally block.

4. **--cron mode (D-08, SCHED-01):**
   - Create `CrontabManager(lock_path=get_data_dir() / "crontab.lock")`
   - Try `manager.schedule(command, cron_expr)`
   - Catch `ValueError` (invalid cron): print red error, exit 1
   - Catch `FileNotFoundError` (navigator not on PATH): print red error, exit 1
   - Catch `TimeoutError` (lock): print red error, exit 1
   - On success: `console.print(f"[green]Scheduled '{command}' with cron: {cron_expr}[/green]")`

5. **--remove mode (D-09, SCHED-03):**
   - Create `CrontabManager(lock_path=get_data_dir() / "crontab.lock")`
   - `removed = manager.unschedule(command)`
   - If removed: green success message
   - If not removed: yellow "Command '{command}' was not scheduled"

Follow established patterns: lazy imports inside function body, Rich Console for output, connection close in finally blocks.
