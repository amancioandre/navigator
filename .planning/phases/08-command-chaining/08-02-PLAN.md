**Part A: Update exec_command in cli.py to follow chains automatically**

In the `exec_command` function, after the existing `execute_command()` call:
- Check if `cmd.chain_next is not None`
- If so, lazy import `execute_chain` from `navigator.chainer`
- Call `chain_result = execute_chain(conn, cmd, config)`
- Display chain results:
  - Print correlation ID: `console.print(f"[dim]Chain ID: {chain_result.correlation_id}[/dim]")`
  - For each result in chain_result.results, print step status (green=success, red=failure)
  - If chain_result.success is False, raise typer.Exit with last non-zero return code
- If `cmd.chain_next is None`, keep existing single-command execution behavior unchanged

Important: The exec handler must NOT call execute_command separately then execute_chain. Instead, when chain_next exists, call ONLY execute_chain (which handles running the first command AND all subsequent ones). When chain_next is None, call execute_command as before (existing behavior preserved).

Restructure the exec handler:
```python
if cmd.chain_next is not None:
    from navigator.chainer import execute_chain
    chain_result = execute_chain(conn, cmd, config)
    # display chain results with correlation ID
    ...
else:
    # existing single-command execution (unchanged)
    result = execute_command(cmd, config, ...)
    ...
```
